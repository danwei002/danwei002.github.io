My Projects

Discord Playing Cards Bot
Cards Bot

A Discord bot I am working on that allows users to play various card games, such as President, with each other in a Discord server. The bot is written in Python using the
discord.py library. User data is stored and accessed using a MySQL database.

The bot currently supports Texas Hold 'Em and President. It also includes a betting system as well as a basic economy system to go with it. The prefix for this bot is 'c!'.

Some examples of commands (full list available under command 'c!help'):

  • c!bal: Check user balance
  • c!hand: Check the cards in your hand
  • c!game: Start a card game in the channel this command was sent
  • c!next: Deal the next community card in Texas Hold 'Em
  • c!play: Play cards by their displayed index in a game of President

Battleship Inspired Game
Grid-Based, Battleship-Inspired Game

A grid-based game inspired by the classic Battleship board game, but with extra twists. I created this with a friend of mine as our final project for our Grade 12 Computer Science course. It is written in Java using the Greenfoot IDE.

In this version of Battleship, there are three types of ships. One is a Submarine, which can be toggled between submerged or not. Submerged submarines cannot be damaged by standard missiles, but are vulnerable to underwater torpedoes. Unsubmerged submarines are the opposite. The next is a Cruiser, which has no special ability. The final ship is the Zoomboat, which can move to an adjacent cell on a given player's turn but also forfeits the player's opportunity to use a weapon during their turn.

There are also three types of weapons. The first is the standard Missile (see the first gif), which hits all non-submerged ships in the cell it drops into. The second is the Torpedo, which is able to damage submerged submarines. The final is the Radar, which scans and reveals all enemy Zoomboats in a 5x5 area around the cell it was used on (see the second gif).

To check out the project for yourself, first install Greenfoot if you haven't already, then download and run the .gfar file from the GitHub repo.

Not Mario Bros - Platformer Game
Not Mario Bros

A game inspired by Nintendo's Mario Bros that I created as the final project for my Grade 11 Computer Science course. The game is written in Java using the Greenfoot IDE. It includes a self-created game engine that handles movement and collision (such as gravity and preventing clipping out of bounds). It is a non-scrolling platformer game consisting of six unique levels designed by myself.

Game Objective: navigate Mario to the flag, which may or may not be initially hidden

Controls: left and right arrow keys to move, hold shift to sprint, spacebar to jump

To check out the project for yourself, first install Greenfoot if you haven't already, then download and run the .gfar file from the GitHub repo.

Battlefield Simulation
Battlefield Simulation

A Clash Royale inspired Battlefield simulation created for one of the group projects in my Grade 12 Computer Science courses. It is written in Java using the Greenfoot IDE.

This simulation is a tower-defense simulation. It simulates a battle between two teams, red and blue. Each team's objective is to destroy the enemy's Hub building. There are three types of buildings: Hubs, Defense Towers (fire at enemy troops), and Artilleries (fire at enemy buildings). There are four types of troops, Infantry (standard), Ninja (high speed and damage, low health), Tanks (low speed and damage, high health), and Medics (heals allied troops).
